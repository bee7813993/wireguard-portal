## 必要なもの

| 項目 | 内容 |
|------|------|
| OS | Windows 10 / 11 |
| ソフトウェア | [WireGuard for Windows](https://www.wireguard.com/install/)（無料） |
| Web サーバー | XAMPP、IIS、nginx など（ポート **80** で動作するもの） |
| ポータル URL | 管理者から共有された WireGuard Portal の URL |

---

## 全体の流れ

```
ポート番号を決める
       ↓
設定ファイル (.conf) を生成・ダウンロード
       ↓
WireGuard にインポート
       ↓
トンネルを有効化
       ↓
ポート 80 で Web サーバーを起動
       ↓
外部から http://[VPSのIP]:[ポート番号] でアクセス可能！
```

---

## STEP 1 — ポート番号を決める

外部からアクセスするときに使う **TCP ポート番号** を決めます。

- 範囲: **1024 〜 65535**
- 例: `11090`
- 同じポート番号は一人しか使えません。他の利用者と重複しないよう管理者に確認してください。

> **注意:** 同じポート番号で再生成すると、以前の設定は無効になります。

---

## STEP 2 — 設定ファイルを生成・ダウンロードする

1. ブラウザで WireGuard Portal を開きます。
2. 「**公開ポート番号**」欄に決めたポート番号（例: `11090`）を入力します。
3. 「**設定を生成する**」ボタンをクリックします。
4. しばらくすると「**クライアント設定**」が表示されます。
5. 「**↓ wg-client_11090.conf をダウンロード**」ボタンをクリックして、ファイルを PC に保存します。

> ダウンロードされるファイル名は `wg-client_[ポート番号].conf` です。

---

## STEP 3 — WireGuard をインストールする

まだインストールしていない場合は、公式サイトからダウンロードしてインストールしてください。

**ダウンロード:** https://www.wireguard.com/install/

インストールは「次へ」を押し続けるだけで完了します。

---

## STEP 4 — トンネルをインポートする

WireGuard を起動すると、最初は以下のような空の画面が表示されます。

> 画面中央に「**ファイルからトンネルをインポート**」ボタンが表示されています。

**手順:**

1. 「**ファイルからトンネルをインポート**」ボタンをクリックします。  
   （または画面左下の「**トンネルの追加 ▼**」→「**ファイルからインポート**」）
2. ダウンロードした `wg-client_11090.conf` を選択して「開く」をクリックします。
3. インポートが完了すると、左側に `wg-client_11090` が表示されます。

インポート直後の画面では、右側に以下の情報が表示されます。

| 項目 | 内容 |
|------|------|
| 状態 | 無効（まだ接続していません） |
| アドレス | `10.100.10.x/24`（自動割り当て） |
| DNS サーバー | `1.1.1.1` |
| ピア / エンドポイント | `[VPSのアドレス]:51821` |
| ピア / Allowed IPs | `10.100.10.1/32` |

---

## STEP 5 — トンネルを有効化する

1. 左側のリストで `wg-client_11090` を選択します。
2. 右側の「**有効化(A)**」ボタンをクリックします。

接続に成功すると:

- 左側のアイコンが **緑色のシールドマーク** に変わります。
- 状態が「**有効**」になります。
- しばらくすると「**直近のハンドシェイク: ○秒前**」が表示されます。
- 「**転送: ○ B 受信済み、○ B 送信済み**」のように通信量が表示されます。

> ハンドシェイクが表示されない場合は、VPS サーバーとの通信ができていません。[トラブルシューティング](#トラブルシューティング) を参照してください。

---

## STEP 6 — Web サーバーを起動する

トンネルが有効になったら、PC 上でポート **80** で Web サーバーを起動します。

### XAMPP を使っている場合

1. XAMPP Control Panel を開きます。
2. 「Apache」の行の「**Start**」ボタンをクリックします。
3. ステータスが緑色になれば起動完了です。

### その他の Web サーバー

| サーバー | 起動確認方法 |
|----------|--------------|
| IIS | 「インターネット インフォメーション サービス (IIS) マネージャー」から起動 |
| nginx | `nginx.exe` を実行 |
| Python | `python -m http.server 80` |

> **重要:** Web サーバーはポート **80** で待ち受けている必要があります。別のポートの場合は設定を変更してください。

---

## STEP 7 — 外部からアクセスして確認する

ブラウザで以下の URL にアクセスします：

```
http://[VPSのIPまたはドメイン]:[ポート番号]
```

例:
```
http://203.0.113.1:11090
```

スマートフォンや別の PC からアクセスすることで、外部公開が正常に動作していることを確認できます。

## トラブルシューティング

### ハンドシェイクが表示されない（接続できない）

| 確認項目 | 対処方法 |
|----------|----------|
| WireGuard のトンネルが有効になっているか | 「有効化(A)」ボタンを押す |
| ファイアウォールが WireGuard をブロックしていないか | Windows Defender ファイアウォールで WireGuard を許可する |
| 設定ファイルが正しいか | ポータルで再生成してインポートし直す |
| 既存のトンネルと設定が混在していないか | 古いトンネルを削除してから再インポートする |

### 外部からアクセスしても繋がらない（nginx の画面が出るなど）

- WireGuard のトンネルが「有効」になっていることを確認してください。
- Web サーバーがポート **80** で起動していることを確認してください。
- 設定を再生成（ポータルで同じポート番号を再入力）してから、WireGuard にインポートし直してください。

### 設定を再インポートする手順

1. WireGuard で古いトンネルを選択 → 左下の「**×**」で削除
2. ポータルで同じポート番号を再入力して「設定を生成する」
3. 新しい `wg-client_XXXX.conf` をダウンロード
4. WireGuard に再インポートして有効化

### ポート番号を変えたい

新しいポート番号でポータルにアクセスして設定を生成し、新しい `.conf` ファイルをインポートしてください。  
古いポート番号の設定は管理者に削除を依頼してください。

## よくある質問

**Q. 複数のポートを使いたい**  
A. 複数の設定ファイルをインポートできますが、同時に有効化できるのは通常 1 つです。用途ごとに切り替えて使用してください。

**Q. PC を再起動したらトンネルが切れた**  
A. WireGuard を起動して「有効化(A)」を押してください。自動起動にしたい場合は、WireGuard の設定画面で「**コンピューターの起動時にトンネルを有効化**」にチェックを入れてください。

**Q. トンネルを有効にすると他のインターネット通信に影響しますか？**  
A. `AllowedIPs = 10.100.10.1/32` の設定では、VPN サーバー宛の通信のみがトンネルを通ります。通常のインターネット通信には影響しません。

**Q. ポート番号を忘れた**  
A. 管理者に問い合わせるか、`.conf` ファイルのファイル名（例: `wg-client_11090.conf` → ポート `11090`）で確認できます。
